What is ART?

ART is a standalone, browser-first framework for building LLM-powered intelligent agents capable of complex reasoning, planning, and tool usage. It provides the building blocks for sophisticated agent systems that can run entirely client-side or with server integration.

Browser-First Design

Built to run fully in the browser using WebAssembly for local LLM inference, enabling privacy-preserving and offline-capable applications.

Modular Architecture

Seven specialized subsystems with clear boundaries, enabling mix-and-match capabilities and easy extension with new functionality.

Rich Observability

Detailed typed observation system provides transparent insights into agent operations for debugging, monitoring, and visualization.

Robust Tool Integration

Schema-driven tool system with registry, validation, and secure execution, planned MCP protocol support for thousands of integrations.

Flexible UI Integration

Socket-based publish/subscribe pattern for reactive UI updates with fine-grained filtering and type safety.

Multiple Reasoning Patterns

Support for various agent reasoning patterns (PES, ReAct, CoT) that can be hot-swapped at runtime, plus multi-agent delegation.

Plan-Execute-Synthesize Pattern

The default execution flow in ART follows the Plan-Execute-Synthesize (PES) pattern, which separates reasoning into three distinct phases:

  • Plan: The agent analyzes the input and creates a structured plan with tool calls.
  • Execute: The planned tool calls are executed in sequence.
  • Synthesize: The agent creates a final response based on the tool execution results.

This pattern provides robust error handling, observability, and separation of concerns.

Alternative Patterns

ART also supports other reasoning patterns that can be selected or even hot-swapped at runtime:

  • ReAct Pattern: Interleaves reasoning and acting in iterative cycles for dynamic adaptation.
  • Chain of Thought (CoT): Focuses on generating explicit step-by-step reasoning.
  • Custom Patterns: Developers can create specialized patterns for specific use cases.

The flexible Agent Core makes it easy to switch between patterns based on your needs.

Schema-Registry-Executor Pattern

ART implements a robust tool integration system based on three key components:

  • Tool Schema: Defines the tool interface using JSON Schema for LLM understanding and runtime validation.
  • Tool Registry: Manages registration and discovery of available Tool Executors.
  • Tool Executor: Implements the actual tool logic with input validation and error handling.

This approach provides clear contracts, strong validation, and secure execution. Each tool call is tracked via the Observation System, providing full transparency into tool operations.

MCP Integration (Planned)

Future versions will integrate with Anthropic's Model Context Protocol (MCP), enabling thousands of tool integrations through a standardized interface. The WebAssembly-based MCP servers will allow these integrations to run client-side for privacy and performance.

LLM Interaction Layer

The Reasoning System manages interactions with Large Language Models through several components:

  • Reasoning Engine: Abstract interface for LLM interaction with unified call API.
  • Prompt Manager: Constructs context-aware prompts for different reasoning phases.
  • Output Parser: Extracts structured data from raw LLM responses.
  • Provider Adapters: Concrete implementations for specific LLM APIs or local models.

Plans for future versions include:

  • LiteLLM-inspired Unified API: Single interface for many LLM providers.
  • Hybrid Inference: Support for Ollama minions architecture.
  • Speculative Decoding: Performance optimization using smaller models to draft content.
  • WebAssembly Inference: Local model execution for privacy and offline use.

Typed Interface Sockets

The UI System provides standardized, typed sockets using a publish/subscribe pattern that decouples the framework's internal events from the application's UI implementation:

  • Typed Socket Interface: Generic design that supports filtering and type safety.
  • ObservationSocket: Emits Observation objects filterable by ObservationType.
  • ConversationSocket: Emits ConversationMessage objects filterable by MessageRole.
  • StateSocket: Optional socket for agent state changes.

This approach enables reactive UI development by allowing components to subscribe only to the events they care about, with strong typing for robust integration.

// Example React hook using the socket system function useArtObservations(threadId, typeFilter) { const [observations, setObservations] = useState([]); const artInstance = useArt(); useEffect(() => { const socket = artInstance.uiSystem.getObservationSocket(); const unsubscribe = socket.subscribe((newObs) => { setObservations(prev => [...prev, newObs]); }, typeFilter); // Subscribe only to specific type return unsubscribe; }, [artInstance, threadId, typeFilter]); return observations; }

Unified Persistence Layer

The Storage System provides a consistent API for persisting and retrieving all framework data, including conversation history, observations, agent state, and vector embeddings:

  • Storage Adapter Interface: Common API regardless of backend.
  • Specialized Repositories: Type-specific data access.
  • Browser-Native Storage: IndexedDB adapter for client-side persistence.
  • Vector Storage: Support for embeddings and semantic search.

The modular design allows for different storage backends while maintaining a consistent API. Future versions will add WebAssembly-based vector stores and graph databases for powerful client-side RAG capabilities.

Multi-Agent Architecture

ART's approach to multi-agent functionality has two complementary aspects:

  • Runtime Pattern Swapping: The Agent Core can dynamically switch between reasoning patterns (PES, ReAct, CoT) at runtime, allowing flexible adaptation to different tasks.
  • Specialized Agent Delegation: The primary Agent Core can delegate tasks to specialized agents with domain-specific knowledge and capabilities.

Future development will expand these capabilities with:

  • Agent Collaboration Protocols: Standardized communication between agents.
  • Cross-Agent Memory: Shared state and knowledge between agents.
  • Role-Based Orchestration: Coordinating multiple agents with different roles.
  • Agent Marketplace: Ecosystem of specialized agents for different domains.

WebAssembly Integration

ART is designed with WebAssembly as a first-class citizen, enabling powerful browser-based capabilities:

  • Client-Side Execution: Run agents entirely in the browser without server dependencies.
  • Local LLM Inference: Execute smaller models directly in the browser.
  • WebAssembly-Based Tools: Integrate complex tools that run locally.
  • Client-Side RAG: Implement retrieval-augmented generation without server roundtrips.

Future development focuses on expanding WebAssembly capabilities:

  • WebAssembly MCP Servers: Run Model Context Protocol servers locally.
  • Browser-Based Vector Stores: Efficient vector databases in IndexedDB.
  • Local Embedding Generation: Generate embeddings in the browser.
  • WebAssembly GraphDB: Graph databases for complex knowledge representation.

Ideal Use Cases

ART excels in a variety of scenarios where browser-first operation, UI integration, and modularity are important.

Privacy-Sensitive Applications

Build agents that process sensitive data entirely client-side using WebAssembly, without sending data to external servers. Ideal for healthcare, legal, and financial assistants where data privacy is paramount.

Browser Extensions & Web Apps

Create browser extensions and web applications with embedded AI capabilities that run locally. Perfect for content analysis, research assistants, and productivity tools that enhance browsing experiences.

Developer Tools

Build coding assistants, documentation generators, and development environments with integrated AI capabilities. Leverage the typed socket system for real-time UI updates as agents analyze and generate code.

Offline-Capable Assistants

Develop assistants that can function without internet connectivity using local model inference and storage. Ideal for field work, remote areas, or situations with unreliable connectivity.

Multi-Agent Systems

Create sophisticated applications with multiple specialized agents that collaborate to solve complex problems. The runtime pattern-swapping and delegation capabilities make ART ideal for adaptive, multi-step workflows.

Tool-Rich Applications

Build agents that integrate with numerous tools and APIs using the robust Schema-Registry-Executor pattern. Planned MCP integration will enable thousands of tool connections with minimal effort.

Getting Started

Follow these simple steps to begin building with the ART Framework.

1

Installation

Install the ART Framework using npm or yarn:

npm install art-framework

Or using yarn:

yarn add art-framework
2

Initialize the Framework

Create and configure your ART instance:

import { createArtInstance, PESAgent } from 'art-framework'; const art = await createArtInstance({ agentCore: PESAgent, storageAdapter: new IndexedDBStorageAdapter({ dbName: 'my-agent-db' }), // Additional configuration options });
3

Process User Input

Handle user queries through the ART instance:

const response = await art.process({ query: "Compare the weather in London and Paris today.", threadId: "user-session-123", // Other options }); console.log(response);
4

Connect to UI

Subscribe to events for real-time UI updates:

// In your UI component: const conversationSocket = art.uiSystem.getConversationSocket(); const unsubscribe = conversationSocket.subscribe((message) => { if (message.role === 'AI') { // Update UI with AI response renderAIMessage(message.content); } }); // Later, when component unmounts: unsubscribe();

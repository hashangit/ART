ART Framework Model Management: A Visual User Story

Scenario: Research Assistant AI

Sarah is using a Research Assistant AI powered by the ART Framework. This AI is configured to use multiple providers (OpenAI, Anthropic, Google) and automatically select the most appropriate model based on her queries. The AI should prioritize advanced reasoning capabilities for complex analysis questions, code understanding for technical topics, and vision capabilities for images, while managing costs efficiently.

Initial Setup

// Configure thread preferences with model preferences and cost settings await art.stateManager.updateThreadConfig('research-assistant-thread', { reasoning: { modelPreferences: { defaultProvider: 'anthropic', preferredModels: { [ModelCapability.REASONING]: { provider: 'anthropic', modelId: 'claude-3-7-sonnet-20250219' // Best for complex reasoning }, [ModelCapability.CODE]: { provider: 'openai', modelId: 'gpt-4o' // Strong coding capabilities }, [ModelCapability.VISION]: { provider: 'gemini', modelId: 'gemini-2.0-flash' // Efficient for image analysis } }, allowCrossProviderFallback: true // Can switch providers if needed }, costPreferences: { maxCostPerCall: 0.05, // $0.05 per API call maximum warnCostThreshold: 2.00, // Warn if thread exceeds $2.00 maxTokensPerThread: 200000, // Token limit for entire thread preferCheaperModels: false // Prioritize capabilities over cost }, autoDetectCapabilities: true // Enable automatic capability detection } });

Here's a Python script using matplotlib to visualize the correlation between interest rates and inflation:

import matplotlib.pyplot as plt import pandas as pd import numpy as np from matplotlib.dates import YearLocator, DateFormatter import seaborn as sns # Sample data (you would replace this with actual data) years = pd.date_range(start='2013-01-01', end='2023-01-01', freq='Y') interest_rates = [0.25, 0.25, 0.50, 0.75, 1.50, 2.25, 1.75, 0.25, 0.25, 4.50] inflation_rates = [1.5, 1.6, 0.1, 1.3, 2.1, 2.4, 1.8, 1.2, 7.0, 6.5] # Create DataFrame data = pd.DataFrame({ 'Date': years, 'Interest_Rate': interest_rates, 'Inflation_Rate': inflation_rates }) # Calculate correlation correlation = np.corrcoef(interest_rates, inflation_rates)[0, 1] # Create figure with two subplots fig, (ax1, ax2) = plt.subplots(2, 1, figsize=(12, 10)) fig.suptitle('Interest Rates vs Inflation (2013-2023)', fontsize=16) # Plot 1: Line chart of both metrics over time ax1.plot(data['Date'], data['Interest_Rate'], 'b-', label='Interest Rate (%)') ax1.plot(data['Date'], data['Inflation_Rate'], 'r-', label='Inflation Rate (%)') ax1.set_ylabel('Rate (%)') ax1.set_xlabel('Year') ax1.legend(loc='best') ax1.grid(True, linestyle='--', alpha=0.7) ax1.xaxis.set_major_locator(YearLocator()) ax1.xaxis.set_major_formatter(DateFormatter('%Y')) ax1.set_title('Interest Rates and Inflation Over Time') # Plot 2: Scatter plot with regression line ax2.scatter(data['Interest_Rate'], data['Inflation_Rate']) sns.regplot(x='Interest_Rate', y='Inflation_Rate', data=data, ax=ax2, scatter=False) ax2.set_xlabel('Interest Rate (%)') ax2.set_ylabel('Inflation Rate (%)') ax2.set_title(f'Correlation: {correlation:.2f}') ax2.grid(True, linestyle='--', alpha=0.7) plt.tight_layout() plt.subplots_adjust(top=0.92) plt.show() # Optional: Save the figure # plt.savefig('interest_vs_inflation.png', dpi=300, bbox_inches='tight')

Key Model Management Components Highlighted

1. QueryAnalyzer

Automatically detected the required capabilities from each query:

  • Economics question ‚Üí REASONING capability
  • Python code request ‚Üí CODE capability
  • Image analysis ‚Üí VISION capability

2. ModelRouter

Intelligently routed queries to the most appropriate model based on:

  • Thread-specific preferences (from StateManager)
  • Required capabilities (from QueryAnalyzer)
  • Implemented fallback when Gemini failed

3. CostTrackingService

Managed token usage and costs:

  • Tracked cumulative costs across multiple queries
  • Triggered warning when thread cost threshold was exceeded

4. ModelSocket (UI Integration)

Provided real-time updates about model management events:

  • Model selection decisions
  • Error notifications and fallback attempts
  • Cost/budget warnings

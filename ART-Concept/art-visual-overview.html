ART Framework Model Management

A unified system for managing interactions with multiple LLM providers and models

Architecture Overview

The Model Management components extend the ART Framework's Reasoning System to support multiple LLM providers and intelligent model selection based on capabilities and requirements.

Agent Core (AC)

Orchestrates the agent's execution flow

Reasoning System (RS)

Manages LLM interactions

Other Systems

Tool, Context, Observation, UI, Storage

Model Registry

Catalogs models and their capabilities

Provider Factory

Creates provider adapters

Query Analyzer

Detects required capabilities

Provider Adapters

Interface with LLM APIs

Model Selection

Chooses appropriate models

Model Validation

Verifies capability support

Model Selection Process

1
Check Explicit Model
Look for explicitly specified provider/model in the call options
2
Check Thread Configuration
If no model specified, check thread configuration preferences:
  • Look for capability-specific model preferences
  • Fall back to default provider preference if no capability match
3
Validate Model Capabilities
Verify the selected model supports all required capabilities
4
Find Best Alternative
If validation fails, find best alternative from same provider
5
Try Other Providers
If no suitable model found in same provider, try others (if attemptFallback=true)
6
Return Result or Error
Either proceed with chosen model or throw meaningful error

Model Registry API

registerProvider(provider: string, modelInfos: ModelInfo[]): void
Register a provider with its models
getModelInfo(provider: string, modelId: string): ModelInfo | null
Get information about a specific model
getModelsForProvider(provider: string): ModelInfo[]
Get all models for a specific provider
validateModel(provider: string, modelId: string, requiredCapabilities?: ModelCapability[]): boolean
Check if a model supports the required capabilities
getBestModelForTask(provider: string, requiredCapabilities: ModelCapability[]): string | null
Find the best model from a provider for a set of required capabilities
getFallbackModel(provider: string, originalModelId: string): string | null
Get a fallback model when the original model is unavailable
getDefaultModelForProvider(provider: string): string | null
Get the default model for a provider
getAllModelsForUI(): ModelUIInfo[]
Get all models in a UI-friendly format
getDisplayName(provider: string, modelId: string): string
Get a user-friendly display name for a model
getCapabilityDisplayName(capability: ModelCapability): string
Get a user-friendly display name for a capability
